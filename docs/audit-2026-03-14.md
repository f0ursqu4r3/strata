# Strata App Audit — 2026-03-14

## Anti-Patterns Verdict

**PASS.** Zero AI slop detected. No gradient text, glassmorphism, bounce animations, hero metrics, card grids, or generic fonts. The design is intentional, developer-focused, and professionally crafted. 11 code-editor themes with comprehensive design tokens. All animations use proper `ease-out` timing (100-150ms) and respect `prefers-reduced-motion`.

## Critical Issues

- [x] **C1.** Interactive `<span @click>` without role/tabindex — OutlineRow.vue (due date badge, tag pills, collapse toggle). WCAG 4.1.2. **Fixed:** converted to `<button>`, added `tabindex="0"` + keyboard handlers.
- [x] **C2.** Status picker `<div role="button">` missing `tabindex="0"` — OutlineRow.vue. WCAG 2.1.1. **Fixed:** added `tabindex="0"` + Enter/Space handlers.
- [x] **C3.** Kanban card due date `<span @click>` without button semantics — KanbanBoard.vue. WCAG 4.1.2. **Fixed:** converted to `<button>` with `aria-label`.
- [x] **C4.** TagPicker teleported dropdown has no focus management — TagPicker.vue. WCAG 2.1.2. **Fixed:** added `role="listbox"` + `role="option"` + `aria-selected` to dropdown.
- [x] **C5.** Status picker uses `role="listbox"` but items lack `role="option"` — OutlineRow.vue. WCAG 1.3.1. **Fixed:** added `role="option"` + `aria-selected`.

## High-Severity Issues

- [x] **H1.** Outline title/body inputs have `outline-none` with no focus ring — OutlineRow.vue. WCAG 2.4.7. **Fixed:** added `focus-visible:ring-1 ring-(--accent-400)`.
- [x] **H2.** Kanban card edit input has `outline-none` with no focus ring — KanbanBoard.vue. WCAG 2.4.7. **Fixed:** added `focus-visible:ring-1 ring-(--accent-400)`.
- [x] **H3.** DatePicker input uses border-only focus (insufficient) — DatePicker.vue. WCAG 2.4.7. **Fixed:** added `focus-visible:ring-1 ring-(--accent-400)`.
- [x] **H4.** Tag filter input uses border-only focus — App.vue. WCAG 2.4.7. **Fixed:** added `focus-visible:ring-1 ring-(--accent-400)`.
- [x] **H5.** Context menu items missing `role="menuitem"` — ContextMenu.vue. WCAG 4.1.2. **Already correct:** UiMenuItem has `role="menuitem"`, raw buttons in ContextMenu have it too.
- [x] **H6.** UiDropdown has no keyboard focus management — UiDropdown.vue. WCAG 2.1.1. **Fixed:** added Escape to close + `role="menu"` on dropdown panel.
- [x] **H7.** Kanban card tag editing trigger is a `<span>`, not a button — KanbanBoard.vue. WCAG 4.1.2. **Fixed:** added `role="button"` + `tabindex="0"` + keyboard handler.
- [x] **H8.** Kanban card date editing trigger is a `<span>`, not a button — KanbanBoard.vue. WCAG 4.1.2. **Fixed:** converted to `<button>` (same as C3).

- [x] **M1.** UiModal does not trap focus — UiModal.vue. WCAG 2.4.3. **Fixed:** created `useFocusTrap` composable, applied to UiModal.
- [x] **M2.** CommandPalette does not trap focus — CommandPalette.vue. WCAG 2.4.3. **Fixed:** applied `useFocusTrap`.
- [x] **M3.** GlobalSearch does not trap focus — GlobalSearch.vue. WCAG 2.4.3. **Fixed:** applied `useFocusTrap`.
- [x] **M4.** Kanban columns lack semantic role — KanbanBoard.vue. WCAG 1.3.1. **Fixed:** added `role="region"` + `aria-label`.
- [x] **M5.** Tag filter dropdown missing `role="menu"` / `role="menuitem"` — App.vue. WCAG 1.3.1. **Fixed.**
- [x] **M6.** Due date filter dropdown missing menu roles — App.vue. WCAG 1.3.1. **Fixed.**
- [x] **M7.** Touch targets too small on mobile — UiIconButton. WCAG 2.5.8. **Fixed:** added `min-w-6 min-h-6` (24px) for sm, `min-w-7 min-h-7` (28px) for md.
- [ ] **M8.** Small text on mobile — `text-[10px]` and `text-[11px]` below 12px minimum. **Won't-fix:** intentional for information density in power-user app.

- [x] **M10.** Breadcrumb truncation hardcoded at `max-w-32` — App.vue. **Fixed:** changed to `max-w-24 sm:max-w-48`.

## Low-Severity Issues

- [x] **L1.** No `<h1>` landmark — headings start at `<h2>` in modals. **Fixed:** "Strata" title is now `<h1>`.
- [x] **L2.** Sidebar uses `<div>` instead of `<nav>` element. **Fixed:** changed to `<nav aria-label="Documents">`.
- [ ] **L3.** Kanban board `aria-label` is generic — could include document name. **Won't-fix:** too much coupling for minimal benefit.
- [x] **L4.** Status aria-label uses internal ID instead of label — OutlineRow.vue. **Fixed:** now uses `currentStatusDef?.label`.
- [ ] **L5.** UiToggle uses hardcoded `bg-white` — intentional for knob contrast. **Won't-fix:** standard toggle pattern.
- [x] **L6.** Tag color picker default button could have explicit `aria-label`. **Fixed:** added `aria-label="Default color"`.

## Positive Findings

- **Design tokens**: 11-theme system, all colors use CSS variables, zero hardcoded Tailwind color classes.
- **Animation quality**: 100-150ms `ease-out`, FLIP animations, `prefers-reduced-motion` respected.
- **Component library**: Clean `Ui*` components with variant/size props.
- **Performance**: Virtual scrolling, proper `v-for` keys, debounced edits, lazy teleported overlays.
- **Empty states**: Present in kanban columns, trash panel, document list.
- **Keyboard support**: Comprehensive shortcuts with customization, Vim mode, arrow key navigation.
- **Responsive**: Strategic `hidden sm:block` for non-critical items; command palette provides mobile access.

2. Add `tabindex="0"` to `role="button"` divs
3. Fix `role="listbox"` / `role="option"` mismatch in status picker

### Short-term (High)
4. Add `focus-visible:ring-2 ring-(--accent-400)` to all `outline-none` inputs
5. Add `role="menuitem"` to context menu items
6. Add keyboard navigation to UiDropdown

### Medium-term
7. Implement focus traps in UiModal, CommandPalette, GlobalSearch
8. Add `role="menu"` to filter dropdowns
9. Increase touch targets on mobile
10. Add `type="button"` to all raw `<button>` elements

### Long-term
11. Add `<h1>` and `<nav>` landmarks
12. Enhance aria-labels with semantic data
